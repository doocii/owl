对我的还有一个很大的教训就是wp重装问题，这几年来不知道重装过多少遍wp，像我不喜欢备份的人来说，重装都是相当于重写，文字还可以快速导出导入，图片就麻烦了，所以日后我写文章能用文字表达的就不要用图片，然后安心的等待下一次重装。

<h3>安装流程</h3>
<ul>

<li>压缩包的根目录里面有一个demo.txt，是后台选项的备份，可以直接在后台导入，方便熟悉后台操作。</li>

<h3>验证问题</h3>
<ul>
<li>主题安装激活后，主题会带你到验证界面，假如提示你没权限的页面的话，你可以返回刷新一下，手动进入验证界面，输入密匙验证即可。</li>
<li>正式版的一个密匙可以验证三个域名，当主域名被验证，二级域名是不能再验证的，本地搭建的环境也不能验证的，别问为什么。</li>
</ul>

<h4>Q：假如我想换域名怎么办？</h4>
A：假如已经验证过，那就回到验证界面，点击“停用”按钮，当前域名就会被停用，你可以用到新的域名了，理论上密匙可以无限次切换使用。

<h4>Q：我重装了，但没点停用怎么办？</h4>

<li>代码高亮：
这里要细说一下，这次采用的代码高亮插件不同于island的prettify.js，它采用的是自动识别高亮代码的方式，效果并不是很好。
现在Owl换成了prism.js，官网是：<a href="http://prismjs.com/" target="_blank">http://prismjs.com/</a>，选择它主要是它皮肤多，支持语言多，可以自定义下载，还带有很多扩展。
想现在Owl主题下默认的prism，支持js/css/html/php/sass,支持代码编号，假如你觉得好不满足，可以自己去自定义下载，然后覆盖主题的prism.js和prism.css即可。
当前高亮的格式为：
<pre class="line-numbers language-markup">
<code>
&lt;pre class="line-numbers">
  &lt;code class="language-css">p { color: red }&lt;/code>
&lt;/pre>
</code>
</pre>
注意到没有，它需要两层标签包裹住高亮代码，pre标签的class一般为插件的类名，code的class定义了语言种类。

<h4>Q:我高亮html格式的语法时，页面都错乱了，怎么回事？</h4>

</li>
<li>萤火背景：打开会占用一定的浏览器内存</li>
<li>视频背景：你有本事，你打开看看</li>
<li>音乐播放器：
	打开后按提示安装插件，创建歌单，再在歌单里面创建播放列表，在列表的右边有一个ID数字，把歌单对应的ID数字粘贴到主题选项这里即可。

<h3>其他问题</h3>
<h4>Q：文章缩略图怎么回事？</h4>
A：文章缩略图存在于首页预览的特色图和文章下面的推荐图。每次想到写个文章还要找配图也是够麻烦的，这次是采用随机图来显示，当你没有上传特色图，就随机一张图片，在images/thumbnail/文件夹下面，你可以存放任意张图片，大小在200px*200px左右，然后在主题选项的文章里修改随即图数目即可，还有就是owl主题的内页不再存在特色图。

<h4>Q：怎么自定义皮肤？</h4>
A：这个问题问得好，主题自带的可定义的部分很有限，只有Banner和Body，想要改变其它部件的背景色或者字体颜色什么可以通过以下方式实现。
本次主题的css采用sass编写，什么是sass可以百度一下，只要会css的人都很容易学会sass。
<ol>
<li>安装编译环境ruby  <a href="http://www.w3cplus.com/sassguide/install.html" target="_blank">http://www.w3cplus.com/sassguide/install.html</a>（只需看ruby安装，不要看下面的sass安装）</li>
<li>安装编译工具koala  <a href="http://koala-app.com/index-zh.html" target="_blank">http://koala-app.com/index-zh.html</a></li>
<li>打开koala，把我的主题解压后的文件夹拖放进去，你会看到几个scss格式的文件，选中最重要的一个style.scss，这时右边栏会滑出来，保持默认的几个勾选的，点击编译按钮即可</li>
<li>刚刚已经编译过一次了，以后你每改动一下scss文件，你的style.css会跟着自动变化，前期工作已经做完，下面是开始动工了</li>
<li>打开你心爱的代码编辑器，打开在文件夹sass下的_skin.scss，所以皮肤颜色都在这个里面，并且我都做好了注释，只需改变几个值，整个style.css都会随着变化</li>
<li>改变_skin.scss后，看看style.css有没有变化，不放心的话，就再按一次编译按钮，完成了</li>
</ol>

<h4>Q：怎么开启主题后，后台反应比以前慢了？</h4>
A：这个是因为functions.php里面有一个“后台更新提醒”功能，用于接收主题最新版本的消息通知的json文件，相当于你每次访问后台都会访问这个json文件，假如觉得慢的话，可以去后台把这个代码的add_action钩子注释掉就可以了，但你以后将无法知道版本更新情况而已。
